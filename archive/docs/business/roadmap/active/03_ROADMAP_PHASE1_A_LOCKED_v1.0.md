# SoftFocus — Roadmap (Phase 1 / Phase A)

Version: v1.0  
Status: ACTIVE  
Core Phases: LOCKED  

---

## Scope

This roadmap reflects the state of SoftFocus after
finalizing and locking Phase 1 (Assessment)
and Phase A (Structured Protocol).

These phases form the immutable core of the product.

---

## Phase 0 — Foundation (COMPLETED)

- Repository initialized
- Automation framework established
- Pre-commit and pre-push gates enforced
- CI validation pipeline bootstrapped
- Documentation filing system (active / archive) created

Status: COMPLETE

---

## Phase 1 — Burnout Assessment (LOCKED)

Purpose:
Provide a neutral, structured assessment
for early burnout detection.

Locked elements:
- assessment intent
- question structure
- response scale
- result categories
- interpretation logic

Changes allowed:
- none

Status: FINAL LOCK

---

## Phase A — Structured Protocol (LOCKED)

Purpose:
Translate assessment results into
a non-prescriptive structured intervention.

Locked elements:
- protocol principles
- module boundaries
- UX flow logic
- paid boundary framing

Changes allowed:
- none

Status: FINAL LOCK

---

## Phase B — Localization (NEXT)

Purpose:
Provide language parity without reinterpretation.

Constraints:
- identical structure across languages
- no cultural rewriting
- canonical terminology preserved

Planned languages:
- Russian
- Spanish
- Arabic

Status: PLANNED

---

## Phase C — Conversion Layer (NEXT)

Purpose:
Enable trust-based monetization
without persuasion or urgency.

Constraints:
- no dark patterns
- no upsells
- no emotional pressure

Status: PLANNED

---

## Phase D — Release & Distribution (FUTURE)

- Tag-based releases only
- Web-first validation
- PWA wrapper
- App Store / Google Play submission

Status: FUTURE

---

## Non-Goals

- Coaching
- Therapy
- Community
- Engagement mechanics
- Gamification

---

## Summary

SoftFocus progresses by locking core logic early
and expanding only through new, explicit phases.

Stability precedes scale.
