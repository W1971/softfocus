# SoftFocus — TEST LOGGING & EXECUTION FLOW

Version: v1.0  
Status: ACTIVE  
Audience: Core maintainers

---

## Цель

Зафиксировать **прозрачную, воспроизводимую и управляемую систему тестирования governance**.

---

## Основные компоненты

### 1. Человекочитаемый лог
- scripts/audit/append-run-log.sh
- tests/runs/<RUN_ID>/run.log
- линейный, append-only

### 2. Машинный лог (JSONL)
- scripts/audit/append-run-jsonl.sh
- tests/runs/<RUN_ID>/run.jsonl
- источник для дашбордов и аналитики

---

## RUN_ID

Каждый запуск:
- имеет уникальный RUN_ID
- создаёт одну директорию
- никогда не перезаписывается

---

## Тестовые сценарии

Сценарии:
- являются shell-скриптами
- обязаны писать оба лога
- не принимаются без логирования

Пример:
- TS-READY-BASE-001

---

## Проверка логов

Скрипт:
- tests/helpers/verify-test-logs.sh

Он гарантирует:
- наличие run.log
- наличие run.jsonl

---

## Инварианты

- append-only
- отсутствие логов = нарушение governance
- тесты не модифицируют прошлые RUN’ы
- решения всегда трассируемы

---

## Результат

Система позволяет:
- видеть историю решений
- анализировать риск со временем
- доказывать управляемость релизов
- автоматизировать контроль без потери человеческого надзора
