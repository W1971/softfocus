# Anti-Requirements: Mobile UI

This document defines what the SoftFocus mobile UI
must explicitly avoid.

---

## Prohibited elements

- Numeric scores or percentages
- Charts, graphs, or visual analytics
- Comparative language (average, above/below)
- Calls to action (imperative phrasing, directive language)
- Gamification or progress indicators
- Multi-step flows on the result screen

---

## Prohibited interaction patterns

- Mandatory scrolling to understand the result
- Expand/collapse as primary interaction
- Modal dialogs for explanations
- Notifications or prompts after results

---

## Prohibited tone

- Motivational language
- Urgency framing
- Performance framing
- Outcome promises

---

## Desktop patterns not allowed on mobile

- Dashboards
- Sidebars
- Tooltips
- Multi-column layouts

---

## Enforcement rule

If a mobile result screen requires explanation,
it violates this policy.

Understanding must be immediate.
