# SoftFocus — Точка входа Governance

Version: v1.0  
Status: ACTIVE  
Audience: Core maintainers, auditors, CI systems  
Locale: RU  

---

## Назначение

Данный документ определяет **каноническую точку входа governance**
в проекте SoftFocus.

Он описывает:
- какие скрипты считаются управляющими;
- в каком порядке они выполняются;
- какие проверки являются обязательными;
- какие состояния блокируют продвижение.

---

## Каноническая точка входа

Единственной допустимой точкой входа governance является:

scripts/governance/full-governance-run.sh    
Ни один скрипт не имеет права:
- обходить данный entrypoint;
- частично выполнять governance-проверки;
- инициировать релиз вне данного потока.

---

## Линейный протокол

Governance выполняется **строго линейно**:

1. Очистка проекта (project-cleanup.sh)
2. Аудит структуры (structure-audit.sh)
3. Выполнение тестовых сценариев
4. Проверка логов (verify-test-logs.sh)
5. Классификация READY
6. Принятие решений (при необходимости)
7. Разрешение или блокировка релиза

Пропуск любого шага **запрещён**.

---

## Обязательные инварианты

- Все шаги детерминированы.
- Все решения логируются.
- Все RUN имеют уникальный RUN_ID.
- Human Override допускается только при LOW RISK.
- STRUCTURAL FAIL всегда блокирует продвижение.

---

## Использование в CI

CI обязан:

- вызывать только canonical entrypoint;
- интерпретировать результат governance;
- блокировать релиз при несоответствии READY.

Любая попытка обойти governance
считается нарушением архитектурных требований.

---

## Финальное положение

Данный документ является
**обязательным нормативным актом governance SoftFocus**.

Изменения допускаются
только через новую версию документа
и пересмотр связанных скриптов.
