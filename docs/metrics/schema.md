# SoftFocus Metrics Schema

This schema defines non-behavioral, non-manipulative metrics.
The goal is understanding usage quality, not optimization pressure.

## Allowed Metrics

### Assessment Completion
- Assessment started
- Assessment completed

Purpose:
Validate clarity and flow, not conversion.

---

### Result Distribution
- Stable Recovery
- Compensated Load
- Recovery Debt
- Systemic Deficit

Purpose:
Understand population-level alignment patterns.

---

### Return Signal Interaction
- Return signal displayed
- Reassessment after return window

Purpose:
Validate timing logic, not retention loops.

---

### Context Variant
- Default
- Founder / high-load

Purpose:
Understand narrative resonance.

## Explicitly Forbidden Metrics
- Time pressure metrics
- Streaks
- Frequency incentives
- Daily active usage targets
- Emotional response tracking

## Principle
Metrics must not alter user behavior.
