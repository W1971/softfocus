# Recovery → Product Bridge

Этот документ формализует переход
от результата калькулятора восстановления
к платному продукту SoftFocus.

## Источник сигнала

Источник:  
`content/kalkulyatory/kalkulyator-vosstanovleniya/thresholds.json`

Используем поле:
- `product_signal`

## Сопоставление сигналов

| product_signal | Действие |
|---------------|----------|
| none          | Продукт не предлагается |
| optional      | Мягкое упоминание продукта |
| recommended   | Явная рекомендация |
| immediate     | Прямой продуктовый переход |

## Целевой продукт

- ID: `anti-burnout-system-30d`
- Тип: цифровой продукт
- Формат: PDF / Notion / инструкции
- Назначение: структурное восстановление при риске выгорания

## Принцип

Калькулятор:
- не продаёт
- не убеждает
- не мотивирует

Он **фиксирует операционное состояние**  
и передаёт управление продуктовой системе.

## Governance

- Bridge является обязательным контрактом
- Изменение порогов требует обновления bridge
- Используется в CI product-gate
